## Decisions locked this session
- **Pages does NOT support Secrets Store bindings** (spiked). Tenant hosting = **Workers + static assets**, not Pages.
- **ADR-036 distribution model:**
  - Repo **stays public for now** — the private flip (M2) is a business paywall decision, not security (no secrets in repo). Deferred indefinitely.
  - **Clients never run lade** — lade operates ALL sessions/cloud envs; clients get only the deployed app bundle in their Cloudflare (+ optionally their own app repo), never lade module source. IP protection is structural. See memory [[clients-never-run-lade]].
  - **Module placement rule:** new module defaults to lade-server (engine API) unless it must run in a lade-operated session (skills/hooks/pack).
  - **CarboNet (licensee) distribution:** deliver core via an auto-synced **mirror repo in their org** (recommended) and/or **vendoring into each app repo** at provision time — NOT hand-minted per-repo tokens. Token automation, if ever, = GitHub App. Deferred to provisioner/reconciler build.
- **Git token is build/session-time, never a runtime secret** — lives in lade keychain + lade cloud envs only, never in a tenant's CF Secrets Store.

## What's blocked & why
- **M2 private flip** — intentionally deferred (ADR-036). Token minted + parked (keychain, `.env.local`, lade cloud env). When ready: paste `CLAUDE_STACK_REPO_TOKEN` into remaining lade-run cloud envs, then flip. Missing an env degrades gracefully (cloud-bootstrap exits 0, session runs stackless).

## Gotchas
- **`.env.local` now also holds `CLAUDE_STACK_REPO_TOKEN`** (read-only, single-repo scoped) alongside live `LADE_*` secrets. Plaintext, gitignored, never tracked. Do NOT commit.
- CF Secrets Store values are **unrecoverable** once stored — save plaintext elsewhere first.
- Provisioner token needs explicit **Secrets Store: Read** account permission — not in default hosting scopes. Account-owned tokens verify at `/accounts/{id}/tokens/verify`.
- **account id ≠ store id** — distinct 32-hex; store id in the `accounts/{id}` slot returns a misleading `10000` auth error, not 404. secret-binder resolves store id live and its error message calls this out.
- secret-binder lib is **bash** — sourcing + running in the zsh session-shell fails (`curl not found` red herring). Run its functions under `bash -c`.
