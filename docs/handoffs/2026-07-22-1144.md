## Gotchas
- **`.env.local` holds LIVE lade secrets** (`LADE_API_TOKEN`, `LADE_SECRET_ACCESS_KEY`)
  in plaintext. Now gitignored (`.env*` rule added this session) and never tracked —
  but it is the provisioner's saved token copy. Consider moving to macOS Keychain
  (`security add-generic-password`) per the secret-binder plan's token-resolution order
  (env → keychain → fail). Do NOT commit it.
- CF Secrets Store values are **unrecoverable** once stored (masked, no API read-back).
  Save plaintext elsewhere before storing, or plan to roll.
- Provisioner token needs the explicit **Secrets Store: Read** account permission —
  NOT in default Pages/Workers/DNS scopes. Account-owned tokens verify at
  `/accounts/{id}/tokens/verify`, not `/user/...`.
- **account id ≠ store id** — a store id in the `accounts/{id}` slot returns a
  misleading `10000 Authentication error`, not a 404. Cost ~1hr this session.
